Project Overview

This project showcases the design of an end-to-end cloud data analytics pipeline that transforms raw retail sales data into analytics-ready datasets for business reporting and decision-making.

Raw data was ingested from Amazon S3 and processed using AWS Glue with PySpark to perform data cleansing, schema standardization, and business-focused transformations. The transformed data was written in optimized Parquet format and loaded into Snowflake for scalable storage and analytics.

Within Snowflake, the data was modeled using a star schema to support efficient analytical queries. SQL-based transformations and views were created to enable downstream consumption by business intelligence tools such as Tableau and Power BI. The pipeline was designed to follow real-world enterprise best practices, emphasizing scalability, maintainability, and performance.

While live BI connectivity was constrained by platform limitations, the project architecture supports visualization through extracted datasets, closely mirroring common production analytics workflows.

Key Highlights

  • Built a cloud-based ETL pipeline using AWS Glue and PySpark

  • Designed and implemented analytical data models in Snowflake

  • Applied star schema modeling for optimized reporting performance

  • Created SQL-based views for BI-ready data consumption

  • Designed with scalability and real-world enterprise patterns in mind

Tools & Technologies

  • ETL: AWS Glue (PySpark)

  • Data Warehouse: Snowflake

  • Querying & Modeling: SQL

  • Visualization: Tableau (extract-based)

Outcome

The project results in a clean, analytics-ready data foundation that can be easily consumed by BI and reporting tools, demonstrating strong data engineering fundamentals and practical analytics experience.
